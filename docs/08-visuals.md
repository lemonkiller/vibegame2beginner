# 第8章 让游戏变好看

> 你做的游戏功能完整了，但看起来像1990年的产物。让我们把它推到2026年。

## 为什么视觉重要？

游戏是给人玩的，人先用眼睛玩，再用脑子玩。一个看起来丑的游戏，哪怕玩法再好，很多人连试都不会试。

好消息是：在2026年，**美术不再是程序员的瓶颈**。AI可以帮你生成图片、设计配色、甚至做简单动画。

### 圆角矩形

把所有方块改成圆角，视觉效果立刻提升：

```lua
-- 普通方块（丑）
love.graphics.rectangle("fill", x, y, w, h)

-- 圆角方块（好看）
love.graphics.rectangle("fill", x, y, w, h, 6)  -- 6是圆角半径
```

### 渐变和光影

```lua
-- 按钮渐变效果：上面亮，下面暗
for i = 0, h do
    local t = i / h
    local r = 0.2 + t * 0.1
    local g = 0.5 + t * 0.15
    local b = 0.8 + t * 0.1
    love.graphics.setColor(r, g, b)
    love.graphics.line(x, y + i, x + w, y + i)
end
```

## 第二层：AI生成美术素材

纯代码虽然够用，但配上图片会让游戏更上一层。2026年，用AI生成游戏素材已经非常方便。

### 推荐工具

| 工具 | 类型 | 适合 | 费用 |
|------|------|------|------|
| Midjourney v7 | AI绘图 | 高质量概念图、角色图 | 订阅制 $10/月起 |
| GPT Image | AI绘图 | 快速出图，对话式生成 | 含在ChatGPT Pro中 |
| Flux 1.1 Pro | AI绘图(开源) | 开发者友好，本地部署 | 免费开源/API按量 |
| Stable Diffusion 3.5 | AI绘图(本地) | 完全控制、免费 | 免费但需GPU |
| Sprite AI | AI精灵生成 | 游戏像素精灵 | 免费/订阅 |
| Aseprite | 像素画编辑器 | 像素画专业工具 | 付费$20 |

**Flux 1.1 Pro** 是目前最强大的开源图像生成模型之一。相比 Stable Diffusion，它在文字渲染、人体结构和解剖准确性方面有显著提升，且支持本地部署和 API 调用。对于游戏开发者而言，Flux 1.1 Pro 的优势在于：可以完全离线运行（无需联网，数据不出本机），支持通过 ComfyUI 等工具搭建自动化流水线批量生成素材，并且开源许可允许将生成的图片直接用于商业项目。如果你有一块 8GB 以上显存的 GPU，优先考虑 Flux 1.1 Pro。

### 游戏专用AI美术工具

通用AI绘图工具生成的图片，往往需要手动裁剪、调整透明度、缩放到像素尺寸，才能在游戏中使用。下面两个工具直接面向游戏开发，生成的素材拿来就能用。

**Sprite AI**（sprite-ai.art）：专为游戏设计的AI精灵生成器。你指定像素尺寸（16x16, 32x32, 64x64 等），它直接生成对应分辨率的游戏精灵图，支持透明背景 PNG 导出。做平台跳跃、RPG、射击游戏的角色和敌人素材时特别省事。免费额度足够小项目使用，量大可订阅。

**SEELE**（seeles.ai）：AI原生游戏开发平台。直接在浏览器内的游戏引擎里生成像素精灵、精灵图集和完整 2D 游戏素材。和 Sprite AI 的区别是，SEELE 不只是生成单张图片，而是提供一整套从素材到游戏的管线——你可以在同一个环境里生成素材、编排动画、搭建关卡。适合想快速出原型的独立开发者。

### 让AI生成游戏素材的提示词

> 我正在做一个2D像素风格的RPG游戏。请帮我生成以下图片描述，我会用这些描述在AI绘图工具中生成素材：
> 1. 主角：一个穿皮甲的年轻冒险者，像素风格，32x32，正面站立
> 2. 村庄背景：中世纪风格小村庄，像素风格，分辨率400x300
> 3. 巨龙Boss：一条红色巨龙，像素风格，64x64

### 在 LÖVE2D 中使用图片

```lua
-- 加载图片（在 love.load 中）
local player_img = love.graphics.newImage("assets/images/player.png")

-- 绘制图片（在 love.draw 中）
love.graphics.draw(player_img, x, y)

-- 缩放绘制
love.graphics.draw(player_img, x, y, 0, scale_x, scale_y)
```

> **注意**：LÖVE2D 支持的图片格式：PNG、JPG、BMP、TGA。强烈建议用 PNG，因为它支持透明通道。

### 图片存放规范

## 第三层：动画

### 帧动画

最简单的动画方式：准备几张图片，按帧切换。

**关键实现思路**：
- 在 `love.load` 中把所有帧图片加载到数组
- 维护一个帧计时器和当前帧号
- 在 `love.update` 中累加计时器，超过帧间隔就切换到下一帧
- 在 `love.draw` 中绘制当前帧图片

> 让AI帮你做更简单：

 > 帮我在 LÖVE2D 中实现角色的4帧行走动画。图片文件是 player_1.png ~ player_4.png，每帧显示0.15秒，循环播放。

### 代码动画

不需要图片也能做动画：

```lua
-- 呼吸效果（大小循环变化）
local scale = 1 + math.sin(love.timer.getTime() * 2) * 0.05

-- 弹跳效果
local bounce_y = -math.abs(math.sin(love.timer.getTime() * 5)) * 10

-- 闪烁效果
if math.floor(love.timer.getTime() * 8) % 2 == 0 then
    -- 显示
end
```

## 第四层：UI美化

游戏的"脸面"——开始界面、暂停菜单、道具栏——这些UI组件决定了玩家对游戏的第一印象。

### 用AI帮你设计UI

> 帮我为 LÖVE2D 游戏设计一个优雅的开始界面：暗色背景，居中大标题，下方两个按钮（开始游戏、退出），有微妙的发光效果和动画。

### 通用UI组件

值得积累的UI组件：

| 组件 | 说明 | 难度 |
|------|------|------|
| 文字按钮 | 可点击的文字按钮，有悬停效果 | ⭐ |
| 图文按钮 | 图标+文字的按钮 | ⭐⭐ |
| 进度条 | HP条、加载条 | ⭐ |
| 面板 | 半透明浮层，用于对话框、背包 | ⭐⭐ |
| 滚动列表 | 物品栏、设置页 | ⭐⭐⭐ |
| 弹出通知 | 获得道具、伤害数字等 | ⭐⭐ |

> 建议：把常用的UI组件做成单独的函数，不同游戏之间可以复用。
