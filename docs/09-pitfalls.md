# 第9章 避坑指南

> 别人踩过的坑，你不用再踩。这份指南是无数次"为什么又不行了"的经验总结。

---

## 坑位一：AI生成的代码跑不起来

### 现象
你把AI给的代码复制进去，运行 `love .`，弹出错误或者窗口闪退。

### 原因
- AI可能给了一个过时版本的 LÖVE API
- AI可能用了 LÖVE 不支持的 Lua 库
- AI可能漏掉了一些依赖的函数

### 解决方案

1. **看错误信息**：终端里会打印错误行号和错误类型。把完整的错误信息复制给AI：

   > 运行游戏时出现以下错误，请帮我修复：
   > ```
   > Error: main.lua:45: attempt to call field 'newGrid' (a nil value)
   > ```

2. **确认 LÖVE 版本**：告诉AI你用的是哪个版本：

   > 我使用的是 LÖVE 11.5，请确保代码兼容这个版本。

3. **分段验证**：不要一次贴一大段代码。先写最小可运行版本，确认能跑，再逐步加功能。

## 坑位二：改完代码效果不对，但没报错

### 现象
代码运行了，窗口出来了，但效果和你想的不一样。比如蛇不移动、球穿过砖块、按键没反应等等。

### 原因
逻辑错误不会产生报错信息，但会让游戏行为不正确。

### 解决方案

1. **用 print 调试**：在可疑的位置加上 `print`，看变量值是否正确：

   ```lua
   print("蛇头位置:", snake[1].x, snake[1].y)
   print("方向:", direction.x, direction.y)
   print("分数:", score)
   ```

2. **描述具体现象**：告诉AI不是"代码有bug"，而是"具体什么行为不对"：

   > 蛇吃到食物后身体没有变长，但分数增加了。请检查碰撞后的蛇身增长逻辑。

3. **回退到上一个能工作的版本**：每改一步都保存一个备份。改出问题时，回到上一个能跑的版本，重新来。

## 坑位三：游戏越做越乱

### 现象
一个 `main.lua` 文件写了1000行，改一个功能其他功能就坏。

### 原因
所有逻辑混在一起，改A影响B。

### 解决方案

1. **拆文件**：按功能拆开

   ```
   main.lua          ← 入口，调用其他模块
   player.lua         ← 玩家相关的逻辑
   enemy.lua          ← 敌人相关的逻辑
   ui.lua             ← UI绘制
   ```

2. **用 local 限制作用域**：

   ```lua
   -- 不好：全局变量，到处都能改
   score = 0

   -- 好：用模块封装
   local M = {}
   local score = 0  -- 外部不能直接访问

   function M.get_score() return score end
   function M.add_score(n) score = score + n end
   return M
   ```

3. **每次只改一个功能，改完就测**。这是铁律。

## 坑位四：AI给你的代码风格不一致

### 现象
AI三次给了你三段代码，变量命名风格不同、缩进不同、逻辑写法也不同。

### 原因
每次回答AI都是重新生成，不会自动保持和你之前的代码一致。

### 解决方案

在提示词中明确代码风格：

> 请使用以下代码风格：
> - 变量名用 snake_case
> - 缩进用4个空格
> - 函数前面加注释说明用途
> - 和我现有代码保持一致的写法

或者让AI重构：

> 帮我统一这段代码的风格，和项目中的其他文件保持一致。以下是项目中的现有代码风格示例：

## 坑位五：性能问题

### 现象
游戏开始很流畅，玩一会儿越来越卡。

### 原因

1. **内存泄漏**：每帧创建新对象，旧的没有回收
2. **过度绘制**：画了屏幕上看不见的东西
3. **低效算法**：每帧遍历了大量数据

### 解决方案

1. **复用对象**：粒子用对象池，不要每帧 `table.insert` + `table.remove`

2. **减少每帧计算**：不变的值算一次存起来

3. **让AI帮你分析**：

   > 我的 LÖVE2D 游戏运行一段时间后变慢了。请在我的代码中找出可能的性能问题：
   > 1. 检查是否有每帧重复创建对象的地方
   > 2. 检查是否有不必要的全量遍历
   > 3. 检查图片加载是否在 update 中（应该在 load 中）

4. **用 LÖVE 的内置统计**：

   ```lua
   -- 在 love.draw 的末尾加上
   love.graphics.setColor(1, 1, 1)
   love.graphics.print("FPS: " .. love.timer.getFPS(), 10, 10)
   love.graphics.print("内存: " .. collectgarbage("count") .. " KB", 10, 30)
   ```

## 坑位六：路径和文件名问题

### 现象
在你的电脑上运行正常，给别人就找不到文件。

### 原因

1. Windows 不区分大小写，Linux 区分：`Image.png` 和 `image.png` 在 Windows 是同一个文件，在 Linux 是两个
2. 用了绝对路径：`C:\Users\xxx\game\...`
3. 中文路径或文件名导致编码问题

### 解决方案

1. **文件名全用小写，用下划线代替空格**
2. **用相对路径**：永远用 `"assets/images/player.png"` 而不是绝对路径
3. **避免中文文件名**

## 坑位七：版本管理混乱

### 现象
改了半天发现改错了，想回到之前能跑的版本，但找不到了。

### 解决方案

用 Git。哪怕你不会 Git 命令，Cursor 集成了 Git 操作：

```bash
# 初始化仓库
git init

# 每完成一个功能，提交一次
git add .
git commit -m "添加了食物生成逻辑"
```

**核心原则**：每次改完一个功能且测试通过，就提交一次。提交信息写清楚了改了什么。

## 坑位八：功能贪多嚼不烂

### 现象
你有一个绝妙的游戏想法，想一次性做完所有功能，结果什么都没做完。

### 解决方案

**MVP（最小可行产品）思路**：

1. 先做一个只能跑的版本——哪怕只有一个方块在屏幕上移动
2. 然后加一个功能——比如方块能跳
3. 再加一个功能——比如有障碍物
4. 每一步都能运行、都能玩

不要追求一步到位。游戏开发是一场马拉松，不是百米冲刺。

## 坑位九：AI上下文丢失

### 现象
你和AI聊了很久，突然AI给出的代码和你之前的完全不一致，好像忘记了之前说的。

### 原因
AI有上下文长度限制，超过限制后早期的内容会被遗忘。

### 解决方案

1. **定期让AI总结当前项目状态**：

   > 请总结一下当前游戏的所有功能和代码结构，我来确认一下。

2. **维护一个开发文档**：把每次修改记录下来，需要时贴给AI

3. **开始新对话时，先提供项目概述**：

   > 我在做一个 LÖVE2D 贪吃蛇游戏，目前功能包括：方向键控制、碰撞检测、分数系统、开始/结束界面。代码在 main.lua 中，以下是当前代码：（粘贴代码）
   >
   > 我现在想添加一个暂停功能。

## 坑位十：沉迷AI依赖

### 现象
什么都问AI，连改一个颜色值都要AI帮忙。离开AI完全写不了代码。

### 解决方案

AI是你的工具，不是你的替代。建议：

1. AI给的代码，至少通读一遍，大致理解它在做什么
2. 遇到简单的修改（改颜色、改数字），试着自己改
3. 每次AI修复了Bug后，想想如果下次遇到类似问题自己能不能修
4. 做完每个游戏后，写一个简短的总结：用了什么技术、踩了什么坑

不需要成为专业程序员，但理解一点代码会让你和AI的合作效率翻倍。

## 坑位十一：图片/音频加载失败

### 现象
运行游戏时弹出错误，常见报错信息：

```
Error: main.lua:12: Could not open file assets/images/player.png. Does not exist.
```

或

```
Error: main.lua:15: Cannot determine file type for assets/music/bgm.ogg.
```

没有图片的画面全是白色方块，没有声音的游戏一片安静。

### 原因

1. **文件路径写错了**：大小写不对、拼错了文件夹名、多了或少了子目录
2. **格式不支持**：LÖVE2D 图片支持 PNG/JPG/BMP/TGA，音频支持 OGG/WAV/MP3/FLAC。放了个 `.webp` 或 `.aac` 进去就会报错
3. **文件本身损坏**：下载或转码过程中文件损坏，能打开但 LÖVE 解析失败
4. **加载时机不对**：在 `love.load` 之外加载资源，导致路径解析出错

### 解决方案

1. **先确认文件存在且路径正确**：

   ```lua
   -- 在 love.load 里先用 io.open 检查文件是否存在
   local f = io.open("assets/images/player.png", "r")
   if f then
       f:close()
       player_img = love.graphics.newImage("assets/images/player.png")
   else
       print("警告：找不到图片 assets/images/player.png")
   end
   ```

2. **确认文件格式受支持**：图片用 PNG（支持透明），音频用 OGG（体积小、质量好）。不要用冷门格式。

3. **所有资源只在 `love.load` 中加载**，不要在 `love.update` 或 `love.draw` 中重复加载：

   ```lua
   function love.load()
       -- 所有资源在这里加载，只加载一次
       player_img = love.graphics.newImage("assets/images/player.png")
       bgm = love.audio.newSource("assets/music/bgm.ogg", "stream")
   end
   ```

## 速查表

| 问题 | 第一反应 |
|------|---------|
| 代码跑不起来 | 看终端错误信息，复制给AI |
| 效果不对但不报错 | `print` 变量值，描述具体现象 |
| 代码太乱了 | 按功能拆文件，用 local 限制作用域 |
| AI前后不一致 | 在提示词中写明代码风格 |
| 越跑越卡 | 检查内存泄漏和每帧重复操作 |
| 路径问题 | 全小写文件名、相对路径 |
| 想回退版本 | 用 Git 管理 |
| 功能太多做不完 | 先做MVP，逐步迭代 |
| AI忘了之前说的 | 定期总结，新对话先提供上下文 |
| 太依赖AI | 通读代码，理解原理，尝试自己改小东西 |
| 图片/音频加载失败 | 检查路径、格式、文件完整性，资源只在 love.load 中加载 |
