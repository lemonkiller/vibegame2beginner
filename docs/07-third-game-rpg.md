# 第7章 第三个游戏：RPG对话冒险

> 前两个游戏里，AI是你的"代码助手"。这个游戏里，AI也是你的"游戏内容"。

## 这个游戏做什么？

一个文字冒险RPG：

- 你是一个冒险者，在幻想世界探索
- 和NPC对话，做出选择，推动故事
- 有简单的战斗系统
- 故事分支由你的选择决定
- 地图由房间组成，可以切换房间

这个游戏和前两个的本质区别是：**内容量大**。贪吃蛇和打砖块的核心逻辑很简洁，但RPG需要大量对话、物品描述、场景描写——这正是AI擅长的领域。

> 📍 RPG对话冒险在复杂度地图的**偏上中间**——系统中等但内容多（大量对话/剧情数据）.

## 新概念：状态机和对话树

### 游戏状态机

RPG比贪吃蛇有更多状态：

```
                    ┌──── 探索 ────┐
                    │              │
               遇到NPC         碰到敌人
                    │              │
                    ↓              ↓
                  对话           战斗
                    │              │
              选择完毕          战斗结束
                    │              │
                    └──── 探索 ────┘
```

### 对话树

NPC的对话不是简单的"说一句话"，而是一棵树：

```
NPC: "你好，陌生人。你是来拯救村子的吗？"
├── 选项A: "是的，我来帮忙"  → NPC: "太好了！先去北边的森林看看吧。"
├── 选项B: "我只是路过"      → NPC: "哦，那你要小心。最近不太平。"
└── 选项C: "拯救村子？出多少钱？" → NPC: "......你这人倒是实在。金币少不了你的。"
```

每个选项可以分支到不同的后续对话，甚至影响后续剧情。这就是"对话树"。

## 创建你的RPG

这个游戏比前两个复杂，建议分步做：

### 第一步：基本框架

> 帮我创建一个 LÖVE2D RPG 框架。要求：
> - 屏幕大小 800x600
> - 顶部3/4是场景区域（描述和图片占位）
> - 底部1/4是对话区域
> - 场景区域显示当前地点名称和描述
> - 对话区域显示当前对话文字，逐字出现（打字机效果）
> - 暂时不需要图片，全部用文字渲染

### 第二步：添加对话选择

> 在之前的RPG框架上添加对话选择系统：
> - 对话文字显示完后，出现2~3个选项
> - 用数字键1/2/3选择
> - 不同选项跳转到不同的后续对话
> - 对话数据用一个Lua表来定义

### 第三步：添加房间切换

> 添加房间系统：
> - 每个房间有名称、描述、可以互动的NPC列表
> - 在探索模式下，用方向键或选项在房间间移动
> - 进入新房间时显示房间描述
> - 房间之间的连接用数据表定义

### 第四步：添加战斗

> 添加简单的回合制战斗系统：
> - 玩家有HP和攻击力
> - 敌人有HP和攻击力
> - 每回合选择：攻击/防御/使用药水
> - 攻击：对敌人造成伤害
> - 防御：下回合受到的伤害减半
> - 药水：回复HP（有限次数）
> - HP归零则游戏结束

### 第五步：完善内容

> 给游戏添加完整的剧情内容：
> - 5个房间（村庄、森林、洞穴、塔楼、王座厅）
> - 4个NPC，每个有3~5轮对话
> - 3种敌人（哥布林、骷髅、巨龙）
> - 最终Boss战
> - 胜利和失败结局

## 完整代码参考

参见 `examples/03-rpg/` 目录。

这个项目的代码比较长，以下是核心结构说明：

### 文件结构

```
03-rpg/
├── main.lua        -- 游戏入口和主循环
├── conf.lua        -- 配置文件
├── game_data.lua   -- 房间、NPC、对话、敌人的数据定义
├── dialogue.lua    -- 对话系统逻辑
├── combat.lua      -- 战斗系统逻辑
└── renderer.lua    -- 绘制函数
```

> 实际完成时，如果AI把所有代码写在一个 main.lua 里也可以。重点不是文件结构，而是游戏能跑起来。

### 对话数据格式

对话数据用Lua表定义，核心结构如下：

- 每个NPC有一组对话，每段对话包含 `text`（对话文字）和 `choices`（选项列表）
- 每个选项包含 `text`（选项文字）和 `next`（跳转到的下一段对话ID）
- 不同选项跳转到不同的后续对话，形成"对话树"

> 完整的对话数据示例和格式说明见[附录C：提示词模板库](appendix-c-prompt-templates.md#RPG)。

## 让AI生成对话内容

RPG最大的工作量不是代码，而是内容。你可以让AI帮你生成大量对话和剧情：

> 请为我的RPG游戏生成一段NPC对话，这个NPC是村庄里的老婆婆，她知道关于巨龙的一个秘密弱点。对话需要3轮，每轮2个选择，不同选择会影响她是否告诉你弱点。

生成后，把对话数据复制到你的游戏代码中即可。

## 变体挑战

RPG的核心技术是"状态机 + 对话树 + 数据驱动"。这三个东西组合起来，能做的游戏种类超乎想象：

| 挑战 | 变化点 | 难度 |
|------|--------|------|
| 🃏 卡牌对战RPG | 战斗改成出牌而非选项 | ⭐⭐ |
| 🔍 解谜冒险 | 战斗改成收集物品+组合解谜 | ⭐⭐ |
| 🏪 商业模拟 | 不打怪，卖东西赚金币进货卖货 | ⭐⭐ |
| 📖 多结局视觉小说 | 无战斗纯故事，5个结局 | ⭐⭐⭐ |
| 🗺️ 随机地牢RPG | 地图每次开始时随机生成 | ⭐⭐⭐ |

**起步提示**：

- 卡牌对战RPG → "战斗界面显示手牌和出牌按钮，每张牌有攻击/防御/特殊效果"
- 解谜冒险 → "去掉战斗，添加物品栏，某些房间需要特定物品组合才能过"
- 商业模拟 → "NPC变成买家和供应商，对话变成讨价还价，金币可以进货"
- 多结局视觉小说 → "去掉战斗系统，对话选项更多分支，设5个结局标记"
- 随机地牢RPG → "房间数据用算法生成而非手写，每次开始地图不同"

> 💡 视觉小说和商业模拟看起来和RPG完全不同，但底层技术一模一样——都是"状态机+数据表+分支选择"。你学的技术比你想的更通用。

## 遇到问题？

| 问题 | 可能的原因 | 解决方法 |
|------|-----------|---------|
| 对话选项没有反应，按数字键1/2/3没效果 | 对话还没播放完（打字机效果还在进行），选项不可选 | 等对话文字全部显示完再按数字键，或者让AI加上"点击跳过打字效果"的功能 |
| NPC不出现或对话内容不对 | 房间数据里NPC列表写错了，或者对话数据的key对不上 | 检查game_data.lua里房间的npc字段和dialogues里的key是否一一对应 |
| 移动按键没反应（比如N/S/D/W切换房间） | 探索模式下按键映射写错了，或者当前不在探索状态 | 确认当前是探索状态（不是对话或战斗中），检查keypressed里的房间切换逻辑 |
| 战斗无法触发 / Boss战进不去 | 可能是前置对话没走完（比如接受任务的节点quest_accept没被触发），导致标志位没设置 | 检查对话树里关键选项的next是否正确跳到了quest_accept节点，确保接受任务后设置了进入战斗的标志 |
| 对话文本显示乱码或截断 | 中文字体没加载，LÖVE2D默认字体不支持中文；或者文本区域宽度不够 | 在love.load里加载一个中文字体（比如系统自带的宋体/黑体），对话区域的换行宽度要设置够大 |
| 游戏卡在某个状态无法继续 | 状态切换的跳转逻辑有死胡同，某个对话选项的next指向了不存在的节点 | 检查所有对话的next指向是否都有对应的节点，确保每个分支都有出路 |
| AI生成的对话内容有格式错误 | AI返回的Lua表格式不对，少了逗号、括号不匹配、或字符串里有未转义的引号 | 把AI生成的对话数据粘贴给AI自己检查，让它"修复这段Lua表的语法错误" |
| 打字机效果太快或太慢 | dt参数没用在计时里，或者每帧追加的字符数不合适 | 调整打字机速度参数，一般每秒显示20~30个字符比较舒服 |
| 对话选项和文字重叠显示 | 选项在对话文字还没显示完时就画出来了 | 加一个判断：只有对话文字全部显示完，才绘制选项列表 |
| 按数字键选了选项但没跳转 | 选项的next字段写错了或为空 | 检查每个选项的next值，确保指向一个存在的对话节点名 |
