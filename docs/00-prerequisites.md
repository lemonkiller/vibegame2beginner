# 第0章 前置认知

> 在开始之前，让我们先搞清楚几件重要的事。

## 你不需要会写代码

这是一个需要反复强调的事实：**在2026年，你不需要会写代码也能做游戏。**

想想你用手机点外卖的过程：你不需要知道外卖APP的后台是怎么写的，你只需要在屏幕上点几下，饭就送到了。现在做游戏也变成这样了——你告诉AI你想要什么，AI帮你把代码写出来。

但这不意味着你什么都不用学。你需要学的是：

| 你需要学什么 | 你不需要学什么 |
|-------------|--------------|
| 用自然语言清楚描述你的想法 | 记忆编程语法 |
| 看到效果后判断"哪里不对" | 从零写代码 |
| 把复杂需求拆成小步骤 | 搭建开发环境 |
| 把错误信息复制给AI帮忙修 | 手动调Bug |

简单来说，你的角色从"写代码的人"变成了"指挥AI写代码的人"。就像从"亲自炒菜"变成了"告诉厨师你想要什么口味"。

## AI能做什么

### 非常擅长的事

- **写小型游戏的完整代码**：贪吃蛇、俄罗斯方块、打砖块这类经典小游戏，AI可以一次性生成可运行的版本
- **帮你理解代码**：把看不懂的代码丢给AI，它能用白话解释给你听
- **修复Bug**：把报错信息复制给AI，它能帮你找到问题并修复
- **生成游戏素材**：AI可以生成像素画、音乐、音效
- **翻译需求为代码**：你用中文说"角色跳起来的时候有个阴影跟着缩放"，AI就帮你写出来

### 能做但需要你盯着的事

- **中大型游戏**：AI可以帮忙写各个模块，但你需要自己把控整体结构和进度
- **性能优化**：AI给出的代码可能不是最优的，需要你测试后反馈问题
- **创意设计**：AI可以给建议，但游戏的"灵魂"还是你的

### 目前做不到的事

- **一键生成3A大作**：AI不是魔法，复杂游戏还是需要大量人工打磨
- **完全理解你的意图**：AI有时候会"听错话"，需要你反复纠正
- **保证代码100%正确**：AI生成的代码可能有问题，需要你测试验证
- **替代你对游戏体验的判断**：好不好玩，只有玩了才知道

## 能力边界地图

把AI编程想象成一个能力坐标轴：

```
                    复杂度高
                       │
     大型RPG │        │  3D开放世界
     策略游戏 │        │  MMO
              │        │
  ────────────┼──────────────── 独立开发难度高
              │        │
   贪吃蛇     │        │  Unity小游戏
   打砖块     │        │
   2048      │        │
                    复杂度低
   AI完全胜任 ←───────→ 需要人工深度参与
```

**经验法则**：如果你能用一段话把游戏规则说清楚，AI大概率能帮你做出来。如果规则说明需要好几页纸，那你需要学着把大项目拆成小步骤，一步一步让AI帮你做。

## 2026年AI编程游戏开发生态

### 三种路线

```
路线A：零代码平台           路线B：AI编程助手           路线C：游戏引擎+AI
─────────────────     ──────────────────     ──────────────────
Rosebud               Pi Coding Agent          Godot + AI
SEELE                 Crush (原OpenCode)       Unity + AI Copilot
GPT Games             Cursor                  Unreal + AI
                      Windsurf
─────────────────     ──────────────────     ──────────────────
纯对话出游戏           边写代码边帮你            专业引擎+AI辅助
零学习成本             需要一点概念             需要学引擎基础
灵活度低               灵活度中               灵活度高
适合快速原型             适合持续迭代           适合正式项目
```

**本手册选择路线B**，使用 LÖVE2D + AI编程助手的方式。原因：

1. 你能看到AI生成的代码，理解原理，而不是黑箱操作
2. 灵活度够高，从小游戏到中等复杂度的2D游戏都能做
3. AI对Lua的代码生成质量很高
4. 后期如果想走路线C，学到的知识可以直接迁移

### 大模型能力概览（2026年4月）

2026年的编程大模型可以分为三档：

- **第一梯队**：Claude Mythos Preview / Opus 4.5、GPT-5.4——编程最强，但需付费
- **第二梯队**：Gemini 3.1 Pro——编程很强，有大量免费额度，上下文最长（1M）
- **开源阵营**：GLM-5.1（中文最强）、DeepSeek V3.2（性价比之王）、Qwen 3.5 Max（阿里出品）、GPT-5.3 Codex——都可免费使用，可通过 Ollama 本地运行

> 数据来源：BenchLM.ai 2026年4月快照。模型能力更新很快，以上仅作参考。详细对比表见[附录D：工具对照表](appendix-d-tool-comparison.md)。

## 一个真实的预期

完成这本手册后，你可以：

- 用AI编程助手独立做出贪吃蛇、打砖块、RPG对话游戏等2D游戏
- 把游戏发布到 itch.io 让全世界的人玩到
- 理解基本的游戏开发概念（游戏循环、碰撞检测、状态管理）
- 独立排查和修复常见问题
- 知道什么时候该问AI、什么时候该自己学一点代码

完成这本手册后，你可能还不能：

- 做出3A级别的游戏（这需要专业团队）
- 完全不看代码就做出复杂系统（复杂项目需要你读一点代码）
- 一次性就做出完美的游戏（迭代是游戏开发的核心）

**但这些都不重要**——重要的是你可以从今天就开始做你的第一个游戏了。
