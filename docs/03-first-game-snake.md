# 第3章 第一个游戏：贪吃蛇

> 2小时，从零到可玩。你的人生第一个游戏即将诞生。

## 这个游戏做什么？

贪吃蛇——那个在诺基亚手机上让无数人沉迷的经典游戏：

- 蛇在屏幕上移动，用方向键控制
- 吃到食物，蛇变长，分数加1
- 撞到墙壁或自己的身体，游戏结束
- 简洁的开始界面和游戏结束界面

别小看这个小游戏。做完它，你就掌握了游戏开发的核心理念：**游戏循环**。

## 什么是游戏循环？

所有游戏都在做同一件事，反复循环：

```
┌─────────────────────────────────────┐
│            游戏循环                   │
│                                     │
│   1. 处理输入  ← 读取键盘/鼠标/手柄   │
│   2. 更新状态  ← 移动角色、检测碰撞   │
│   3. 绘制画面  ← 把当前状态画到屏幕上  │
│         │                           │
│         └──────→ 回到第1步 ─────→    │
└─────────────────────────────────────┘
```

每秒这个循环跑60次（60帧），玩家看到的就是流畅的游戏画面。

在 LÖVE2D 里，这个循环长这样：

```lua
-- LÖVE2D 的游戏循环
function love.update(dt)      -- 第2步：更新状态
    -- 移动蛇、检测碰撞
end

function love.draw()          -- 第3步：绘制画面
    -- 画蛇、画食物
end
```

第1步（处理输入）在 LÖVE2D 里用回调函数来做：

```lua
function love.keypressed(key)  -- 第1步：处理输入
    -- 改变蛇的方向
end
```

**这就是你需要知道的全部游戏开发基础。** 剩下的都是在这个框架上填充具体内容。

## 第一步：创建项目

1. 创建一个文件夹，比如 `snake`
2. 在文件夹中创建 `main.lua` 文件
3. 用 Cursor 或文本编辑器打开

如果你用 Cursor，直接在 AI 对话中输入：

> 帮我创建一个 LÖVE2D 贪吃蛇游戏。要求：
> - 用方向键控制蛇的移动方向
> - 蛇吃到食物后身体变长，分数加1
> - 蛇撞到屏幕边缘或自己的身体就游戏结束
> - 有开始界面（显示游戏标题和"按空格开始"）
> - 有游戏结束界面（显示最终分数和"按R重新开始"）
> - 用网格绘制，蛇是绿色的方块，食物是红色的方块
> - 屏幕大小 800x600，网格大小 20x20
> - 所有代码写在一个 main.lua 中

AI会给你一整段代码。复制到 `main.lua` 中，在终端运行：

```
love .
```

如果一切正常，你会看到贪吃蛇的开始界面！

## 第二步：理解代码

不管AI给你什么样的代码，核心结构一定是这样的：

### 游戏状态

```lua
-- 游戏有哪些"状态"
local state = "start"  -- "start" 开始界面 / "playing" 游玩中 / "gameover" 游戏结束
```

游戏就像一台状态机，同一时刻只处于一个状态。不同状态下，画面和操作逻辑不同：

| 状态 | 画面 | 按键效果 |
|------|------|---------|
| start | 标题 + 提示 | 空格 → 开始游戏 |
| playing | 蛇 + 食物 | 方向键 → 控制蛇 |
| gameover | 分数 + 提示 | R → 重新开始 |

### 蛇的数据

```lua
local snake = {
    body = {{x=10, y=10}},  -- 蛇的身体，每个元素是一个格子坐标
    dir = {x=1, y=0},       -- 当前移动方向
    speed = 10,              -- 移动速度（每秒移动几格）
}
```

**蛇的身体是一个列表**：第一个元素是蛇头，后面是蛇身。蛇移动的时候，蛇头往方向走一格，尾巴去掉一格；吃东西的时候，尾巴不去掉，蛇就变长了。

### 食物

```lua
local food = {x=15, y=15}  -- 食物在网格中的位置
```

食物就是一个坐标。蛇吃到它后，在随机位置生成新的食物。

### 分数

```lua
local score = 0  -- 吃一个食物加1分
```

## 第三步：运行和测试

运行游戏后，依次检查：

1. **开始界面**：是否显示了标题和提示？按空格能否进入游戏？
2. **蛇的移动**：方向键能否控制蛇？蛇是否会自动前进？
3. **吃食物**：蛇碰到食物是否变长？分数是否增加？食物是否重新出现？
4. **游戏结束**：蛇撞墙是否结束？撞自己是否结束？
5. **重启**：按R能否重新开始？分数是否归零？

**每一步如果不对，立刻告诉AI修**。比如：

> 蛇吃完食物没有变长，请检查碰撞检测逻辑并修复

或者：

> 方向键按下后蛇没有改变方向，请修复按键处理

## 第四步：微调

基本功能跑起来后，可以用提示词微调细节：

### 调整速度

> 蛇移动太快了，把速度降低到每秒5格

### 改变颜色

> 把蛇的颜色改成渐变绿色（蛇头深绿，身体浅绿），食物改成黄色圆形

### 添加网格线

> 在游戏区域画淡灰色的网格线，让玩家看清格子

### 添加音效（第8章详细讲）

> 蛇吃到食物时播放一个简短的"叮"声，游戏结束时播放一个低沉的"嘟"声。先用代码生成音效，不使用外部文件。

## 你学到了什么

做完贪吃蛇，你已经掌握了：

| 概念 | 在贪吃蛇中的体现 |
|------|-----------------|
| 游戏循环 | 每帧更新蛇的位置，重新画屏幕 |
| 状态管理 | 开始/游玩/结束三种状态的切换 |
| 输入处理 | 方向键控制蛇的方向 |
| 碰撞检测 | 蛇是否碰到了食物/墙壁/自己 |
| 列表操作 | 蛇身体的增加和减少 |
| 随机数 | 食物的随机位置 |

这些概念是所有游戏的基础。不管你以后做什么游戏，都是这些概念的排列组合。

## 遇到问题？

| 问题 | 可能的原因 | 解决方法 |
|------|-----------|---------|
| 运行后窗口闪一下就关了 | 代码有语法错误 | 看终端的错误信息，复制给AI修 |
| 蛇不移动 | 可能忘了在 update 里更新位置 | 检查 love.update 函数 |
| 蛇吃完食物不变长 | 碰撞检测或身体增长的逻辑有问题 | 把代码给AI，说明问题 |
| 方向键没反应 | keypressed 函数可能写错了 | 检查 love.keypressed 是否正确 |
| 窗口大小不对 | 没设置 conf.lua 或 setMode | 添加 conf.lua 或在 love.load 里设置 |
